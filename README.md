# Overview
This project demonstrates a SOC Analyst approach to analyzing a phishing email. The goal is to identify malicious indicators, extract IOCs (Indicators of Compromise), and provide a detailed investigation report.

The analysis includes email header investigation, URL analysis, domain reputation checks, and threat assessment.

# Objectives
Identify phishing indicators in an email
Analyze sender and domain reputation
Investigate malicious URLs and attachments
Extract Indicators of Compromise (IOCs)
Document findings in a SOC-style report

# Risk Assessment
Severity: High
Potential Impact: Credential theft, Unauthorized access, Financial fraud

# Conclusion
This project shows a practical, real-world approach to analyzing phishing emails from a SOC analyst’s perspective. By examining the email content, headers, sender details, and embedded links, several clear signs of phishing were identified, such as spoofed domains, failed email authentication (SPF/DKIM/DMARC), and suspicious URLs.

It also demonstrates how attackers rely on tactics like typosquatting, urgency, and social engineering to trick users into sharing sensitive information. By identifying and documenting Indicators of Compromise (IOCs), this project highlights how security teams can effectively detect, investigate, and respond to phishing threats.
